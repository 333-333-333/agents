# Laws of UX - Quick Reference

## Checklist for UI Review

### Interaction
- [ ] Touch targets ≥ 48dp (Fitts's Law)
- [ ] Response time < 400ms or loading indicator (Doherty Threshold)
- [ ] Limited choices per screen (Hick's Law)

### Memory
- [ ] Navigation items ≤ 7 (Miller's Law)
- [ ] Content chunked into groups (Chunking)
- [ ] Minimal cognitive load (simple, clear)

### Perception
- [ ] Related items grouped visually (Proximity)
- [ ] Similar functions look similar (Similarity)
- [ ] Groups have clear boundaries (Common Region)

### Behavior
- [ ] Follows platform conventions (Jakob's Law)
- [ ] Primary CTA stands out (Von Restorff)
- [ ] Important items first/last in lists (Serial Position)
- [ ] Progress shown for long tasks (Goal-Gradient)

### Complexity
- [ ] Simplest possible design (Occam's Razor)
- [ ] Complexity hidden, not removed (Tesler's Law)
- [ ] Polished visual design (Aesthetic-Usability)

## One-Line Summaries

| Law | Remember |
|-----|----------|
| Fitts's | Big targets, close placement |
| Hick's | Fewer choices = faster decisions |
| Doherty | < 400ms or show feedback |
| Miller's | 7 ± 2 items max |
| Jakob's | Users expect familiar patterns |
| Von Restorff | Different = noticed |
| Peak-End | Nail the ending |
| Goal-Gradient | Show progress |
| Proximity | Near = related |
| Similarity | Same style = same function |
| Tesler's | Can't eliminate complexity, only move it |
| Occam's | Simplest wins |
